# 🧩 UNIT 2 — OBJECT-ORIENTED DESIGN & UML

---

## 🔹 1. Introduction to OOP

Object-Oriented Programming (OOP) is a paradigm based on **objects and classes**.

### 🎯 Advantages

* Code reusability
* Modularity
* Maintainability
* Scalability

---

## 🔹 2. Classes & Objects

* **Class:** Blueprint/template
* **Object:** Instance of class

**Example:**

* Class → Car
* Object → BMW, Audi

---

## 🔹 3. Four Pillars of OOP

---

### 🔒 Encapsulation

* Wrapping data + methods together
* Restrict direct access

✔ Improves security

---

### 🎭 Abstraction

* Hides internal implementation
* Shows only necessary details

✔ Reduces complexity

---

### 🧬 Inheritance

* One class derives properties from another

✔ Code reuse

---

### 🔁 Polymorphism

* Same function behaves differently

✔ Flexibility

---

## 🔹 4. Access Modifiers

| Modifier  | Access                    |
| --------- | ------------------------- |
| Public    | Accessible everywhere     |
| Private   | Within class only         |
| Protected | Within class + subclasses |

---

# 🔹 5. SOLID Principles (Clean Code Design)

---

### 🧱 S — Single Responsibility Principle (SRP)

> A class should have only **one reason to change**

✔ Keeps code simple

---

### 🔓 O — Open/Closed Principle (OCP)

> Open for extension, closed for modification

✔ Add new features without changing existing code

---

### 🔄 L — Liskov Substitution Principle (LSP)

> Subclass should replace parent without breaking behavior

✔ Ensures correctness

---

### 🧩 I — Interface Segregation Principle (ISP)

> Don’t force classes to implement unnecessary methods

✔ Smaller, specific interfaces

---

### 🔌 D — Dependency Inversion Principle (DIP)

> Depend on abstractions, not concrete classes

✔ Loose coupling

---

# 🔹 6. UML (Unified Modeling Language)

Used to visually represent system design.

---

## 📊 Types of UML Diagrams

---

### 📦 Class Diagram

* Shows classes, attributes, methods
* Relationships between classes

---

### 👤 Use Case Diagram

* Shows interaction between user and system

---

### 🔄 Sequence Diagram

* Shows flow of messages over time

---

### 🔁 Activity Diagram

* Represents workflow/process

---

## 🔗 UML Relationships

---

### 🔗 Association

* Basic relationship between classes

---

### 🔹 Aggregation

* Weak relationship (has-a)
* Example: Team has players

---

### 🔸 Composition

* Strong relationship (part-of)
* Example: Body has heart

---

### 🧬 Inheritance

* Parent-child relationship

---

## 🔹 7. Why OOP + UML Matters
